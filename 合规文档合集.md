## 3. 内容安全策略

### 3.1 内容审核原则
本平台严格遵守《网络安全法》《网络内容生态治理规定》等法律法规，坚持"正能量、主旋律"的内容导向，坚决抵制违法违规内容。

### 3.2 禁止发布的内容
用户不得在本平台发布以下内容：
1. **政治敏感类**：反对宪法确定的基本原则，危害国家安全，泄露国家秘密，颠覆国家政权，破坏国家统一的内容
2. **色情低俗类**：淫秽、色情、赌博、暴力、凶杀、恐怖或者教唆犯罪的内容
3. **侵权类**：侮辱或者诽谤他人，侵害他人名誉、隐私、知识产权等合法权益的内容
4. **虚假信息类**：编造、传播虚假信息，扰乱经济秩序和社会秩序的内容
5. **有害信息类**：宣扬极端主义、民族仇恨、民族歧视，破坏民族团结的内容
6. **其他违法违规类**：违反法律法规、公序良俗的其他内容

### 3.3 审核机制
本平台采用"机审+人审"双层审核机制：
1. **机器审核**：所有用户生成内容（对话、日记、评论等）首先经过AI内容审核系统过滤，识别敏感内容
2. **人工审核**：机器审核疑似违规的内容进入人工审核队列，由审核人员24小时处理
3. **用户举报**：用户可举报违规内容，我们将在24小时内处理反馈
4. **定期巡检**：审核团队定期巡检平台内容，及时发现和处理违规内容

### 3.4 违规处理流程
1. **预警**：机器审核发现疑似违规内容，自动预警并进入人工审核队列
2. **判定**：审核人员根据审核标准判定内容是否违规
3. **处理**：
   - 轻度违规：内容删除，发送警告通知
   - 中度违规：内容删除，限制账号部分功能1-7天
   - 重度违规：内容删除，永久封禁账号
   - 涉嫌违法：保留证据，移交公安机关处理
4. **申诉**：用户对处理结果有异议的，可在7个工作日内提交申诉，我们将在3个工作日内复核

### 3.5 审核团队要求
1. 审核人员需经过法律法规、平台规则、审核标准的专业培训，考核合格后方可上岗
2. 审核团队实行7*24小时轮班制度，确保内容及时处理
3. 建立审核质量抽检机制，确保审核标准执行一致性
4. 审核人员需严格遵守保密制度，不得泄露用户隐私信息

## 5. 危机干预处理流程

### 5.1 适用场景
当用户在对话中出现以下情形时，启动危机干预流程：
1. 明确表达自伤、自杀倾向
2. 表达伤害他人的意图
3. 处于严重情绪崩溃状态，可能发生危险

### 5.2 干预等级
| 等级 | 判定标准 | 响应措施 |
|------|----------|----------|
| 一级（轻度） | 情绪低落、表达负面情绪，但无明确自伤意图 | AI主动安抚，建议寻求支持 |
| 二级（中度） | 明确表达自杀/自伤想法，但无具体计划 | AI立即输出危机干预话术，提供心理援助热线，人工客服介入 |
| 三级（重度） | 有明确自杀/自伤计划，或正在实施危险行为 | AI立即输出干预话术，同时通知紧急联系人（如用户已授权），必要时报警 |

### 5.3 干预流程
```
用户输入 → AI内容检测 → 识别危险内容 → 判定干预等级
    ↓
一级：AI安抚 → 记录案例 → 后续跟踪
    ↓
二级：AI输出干预话术 → 提供全国心理援助热线 → 人工客服1小时内跟进 → 记录归档
    ↓
三级：AI输出干预话术 → 联系紧急联系人（如有授权） → 同时报警 → 上报监管部门 → 记录归档
```

### 5.4 干预话术规范
**核心原则**：不评判、不指责、共情理解、提供支持

**示例话术**：
> "我感受到你现在非常痛苦，你一定承受了很多压力。请你一定不要伤害自己，生命是最宝贵的，还有很多人关心你。
> 
> 如果你现在需要帮助，可以拨打全国心理援助热线：400-161-9995，有专业的心理咨询师24小时为你提供帮助。
> 
> 如果你愿意，可以和我说说你现在的感受，我会一直陪着你。"

## 6. 数据安全管理制度

### 6.1 数据分类分级
| 数据级别 | 定义 | 示例 | 保护要求 |
|----------|------|------|----------|
| 核心数据 | 一旦泄露会造成严重安全风险的数据 | 用户手机号、对话记录、日记内容 | 加密存储、严格访问控制、操作审计 |
| 重要数据 | 泄露会造成较大影响的数据 | 用户行为数据、消费记录 | 访问控制、定期备份 |
| 一般数据 | 公开数据或匿名化数据 | 统计报表、公开知识库内容 | 常规安全防护 |

### 6.2 数据访问控制
1. 实行"最小权限"原则，员工仅获得完成工作必需的数据访问权限
2. 核心数据访问需双人审批，且仅允许在办公网络环境下访问
3. 所有数据访问操作留痕，保存操作日志至少6个月
4. 员工离职时立即回收所有数据访问权限

### 6.3 数据加密
1. 传输加密：所有数据传输采用HTTPS协议，TLS 1.2及以上版本
2. 存储加密：核心数据采用AES-256加密存储，密钥由专人管理，定期轮换
3. 备份加密：备份数据同样加密存储，与生产数据同等保护级别

### 6.4 数据生命周期管理
1. **收集**：仅收集提供服务必需的最少数据，明确告知用户收集用途
2. **使用**：仅在授权用途范围内使用数据，不得超范围使用
3. **存储**：根据数据类型设定保留期限，到期自动删除或匿名化
4. **删除**：用户注销账号后，除法律法规要求保留的交易记录外，其他数据在15个工作日内删除
5. **销毁**：数据销毁采用不可恢复的技术手段，确保数据完全清除

### 6.5 数据安全事件处理
1. 建立数据安全事件应急响应预案，定期演练
2. 发生数据安全事件后，第一时间启动应急预案，采取措施防止事态扩大
3. 按照法律法规要求，及时向监管部门报告，并告知受影响用户
4. 事件处理完成后进行根源分析，完善安全措施，避免类似事件再次发生
