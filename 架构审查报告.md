#### ⚠️ 发现的问题

**问题1: WebSocket消息类型不一致**

- **位置**: `backend/app/websocket/chat.py`
- **描述**: WebSocket发送的消息类型 (`chunk`, `content`) 与前端期望不一致

```python
# 后端发送 (line 239-242)
await websocket.send_text(json.dumps({
    "type": "content",  # ← 应该是 "chunk"
    "content": chunk,
}))

# 前端期望 (chat.tsx line 125)
} else if (data.type === 'chunk') {
    setStreamingMessage(prev => prev + data.content)
```

**问题2: 注册API密码字段名不一致**

- **位置**: `backend/app/api/v1/auth.py` vs `frontend/web/src/api/request.ts`
- **描述**: 后端接收 `password`，前端发送 `password`

```python
# 后端 (auth.py line 68)
if not request.password or len(request.password) < 6:

# 前端 (request.ts line 70)
register: (data: { phone: string; password: string; ... })
```

**问题3: 危机资源API路径**

- **位置**: `backend/app/api/v1/auth.py`

- **建议**: 应使用 `/auth/crisis-resources` (kebab-case)

### 2.2 业务逻辑层审查

#### ✅ 优点

| 项目 | 说明 |
|------|------|
| 服务模式 | 单例模式 `get_xxx_service()` |
| 事务处理 | `db.begin()` / `db.commit()` / `db.rollback()` |
| 重试机制 | LLM调用支持指数退避重试 |
| 危机检测 | 独立 CrisisService |
| 内容过滤 | ContentFilter 多策略支持 |

#### ⚠️ 发现的问题

**问题1: diary_service.py 枚举值处理错误**

- **位置**: `backend/app/services/diary_service.py` line 195-196
- **严重度**: 🔴 高
- **问题代码**:
```python
mood_level_str = self._get_mood_level(request.mood_score)
update_data["mood_level"] = mood_level_str.upper()  # ← 错误！
```

- **原因**: `_get_mood_level()` 已返回小写字符串（如 `"neutral"`），再调用 `.upper()` 会导致枚举值不匹配
- **正确做法**: `update_data["mood_level"] = MoodLevel(mood_level_str)`

**问题2: chat_service.py 数据库事务泄漏**

- **位置**: `backend/app/services/chat_service.py` line 61-206
- **严重度**: 🟡 中
- **问题**: `db.begin()` 未在所有异常路径正确处理 rollback
- **建议**: 使用 `async with db.begin()` 或 `try/except/finally` 确保 rollback

**问题3: i18n 翻译函数实现不完整**

- **位置**: `backend/app/core/i18n.py` line 42-47
- **问题**: `get_translation()` 依赖语言参数，但 `auth.py` 等处调用未传递

```python
# i18n.py
def _(message: str, language: str = 'zh_CN') -> str:

# auth.py 调用
return {"message": _("验证码已发送")}  # ← 未传递 language
```

#### ⚠️ 发现的问题

**问题1: 用户模型缺少邮箱唯一性验证**

- **位置**: `backend/app/models/user.py` line 26
- **问题**: email 字段允许 NULL，但唯一索引在 NULL 值上可能失效

```python
email = Column(String(255), unique=True, index=True, nullable=True)
# SQLite/MySQL: 多个 NULL 值的唯一索引行为不一致
```

**问题2: 敏感信息存储**

- **位置**: `frontend/web/src/stores/index.ts`
- **问题**: Token 存储在 localStorage，存在 XSS 风险

```typescript
localStorage.setItem('access_token', access_token)  // ← 建议使用 httpOnly cookie
```
