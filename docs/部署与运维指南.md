## 快速部署

```bash
# 克隆项目
git clone https://github.com/lcp5674/emotion-ai-assiant.git
cd emotion-ai-assiant

# 配置环境
cp .env.docker.example .env.docker
# 编辑 .env.docker 填入必要的配置

# 一键部署
chmod +x deploy/deploy.sh
./deploy/deploy.sh
```

部署完成后访问：
- 前端：`http://localhost`
- 后端API：`http://localhost/api/v1/health`
- API文档：`http://localhost/docs`

## 部署指南

> 完整内容请查看 `DEPLOYMENT_GUIDE.md`

### 环境准备

#### Docker 安装

**macOS:**
```bash
brew install --cask docker
```

**Ubuntu:**
```bash
sudo apt update
sudo apt install docker.io docker-compose
sudo systemctl start docker
sudo usermod -aG docker $USER
```

### Docker Compose 配置

```yaml
# docker-compose.yml 核心配置
services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=mysql+pymysql://user:pass@mysql:3306/emotion_ai
      - REDIS_URL=redis://redis:6379/0
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy

  mysql:
    image: mysql:8.0
    volumes:
      - mysql_data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping"]
      interval: 10s
      retries: 5
```

### Nginx 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api/ {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

> 完整内容请查看 `PRODUCTION_DEPLOYMENT_GUIDE.md`

### 生产环境要求

| 资源 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2核 | 4核 |
| 内存 | 4GB | 8GB |
| 磁盘 | 50GB | 100GB SSD |
| 带宽 | 5Mbps | 10Mbps |

### SSL 证书配置

```bash
# 使用 Let's Encrypt
sudo certbot certonly --nginx -d your-domain.com

# 复制证书
sudo cp /etc/letsencrypt/live/your-domain.com/fullchain.pem deploy/ssl/cert.pem
sudo cp /etc/letsencrypt/live/your-domain.com/privkey.pem deploy/ssl/key.pem
```

### Nginx HTTPS 配置

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;

    # ... 其他配置
}
```

## 备份与恢复

### 自动备份

```bash
# 配置定时备份 (crontab)
0 2 * * * /path/to/emotion-ai-assiant/deploy/backup.sh

# 备份保留策略：保留最近7天
# 备份位置：./backups/
```

### 手动备份

```bash
# 创建备份
./deploy/backup.sh

# 查看备份列表
ls -la backups/

# 恢复数据库
./deploy/restore.sh backups/emotion_ai_20260411_120000.sql.gz
```

### 数据恢复步骤

```bash
# 1. 停止服务
./deploy/stop.sh

# 2. 恢复数据
./deploy/restore.sh <备份文件>

# 3. 重启服务
./deploy/restart.sh

# 4. 验证
./deploy/health_check.sh
```

## 运维脚本

### 常用命令

```bash
# 部署
./deploy/deploy.sh                    # 一键部署
./deploy/deploy.sh --env=production  # 生产环境部署

# 服务管理
./deploy/start.sh                    # 启动服务
./deploy/stop.sh                    # 停止服务
./deploy/stop.sh --force             # 强制停止
./deploy/restart.sh backend          # 重启后端
./deploy/restart.sh all              # 重启所有服务

# 健康检查
./deploy/health_check.sh             # 简洁输出
./deploy/health_check.sh json        # JSON格式

# 日志
./deploy/logs.sh backend 100         # 查看后端日志
./deploy/logs.sh all 200 true        # 跟随查看所有日志

# 数据库
./deploy/backup.sh                   # 备份
./deploy/restore.sh <文件>           # 恢复

# 清理
./deploy/clean.sh                    # 清理服务
./deploy/clean.sh --volumes          # 清理所有数据
```

### 健康检查 API

```bash
# 检查所有服务
curl http://localhost/api/v1/health

# 检查数据库连接
curl http://localhost/api/v1/health?check=db

# Docker Compose 状态
docker compose ps
```

## 故障排查

### 常见问题

| 问题 | 解决方案 |
|------|---------|
| 后端无法启动 | 检查环境变量配置，执行 `docker logs emotion-ai-backend` |
| 数据库连接失败 | 确认MySQL已就绪，执行健康检查 |
| 前端502错误 | 检查后端服务状态，查看nginx日志 |
| 端口冲突 | 修改 `.env.docker` 中的端口映射 |

### 日志查看

```bash
# 后端日志
docker logs -f emotion-ai-backend --tail=100

# Nginx日志
docker logs -f emotion-ai-nginx --tail=100

# 数据库日志
docker logs -f emotion-ai-mysql --tail=100

# 所有服务日志
./deploy/logs.sh all 100 true
```
