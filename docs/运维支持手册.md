## 日常检查

### 每日检查清单

| 检查项 | 执行频率 | 脚本 |
|--------|----------|------|
| 服务健康 | 每日2次 | `./deploy/health_check.sh` |
| 日志异常 | 每日1次 | `./deploy/logs.sh` |
| 数据库备份 | 每日1次 | `./deploy/backup.sh` |
| 磁盘空间 | 每日1次 | `df -h` |
| 内存使用 | 每日1次 | `free -h` |

### 监控指标

```bash
# CPU使用
docker stats --no-stream

# 内存使用
docker stats emotion-ai-backend

# 网络流量
docker network inspect emotion-ai-network
```

## 故障处理

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 后端无法启动 | 环境变量缺失 | 检查 `.env.docker` |
| 数据库连接失败 | MySQL未就绪 | `docker compose up -d mysql` |
| 前端502 | 后端异常 | `docker logs emotion-ai-backend` |
| 端口冲突 | 端口被占用 | 修改端口映射 |

### 应急响应

1. **发现问题** → 检查健康状态
2. **定位原因** → 查看日志
3. **快速修复** → 重启服务或回滚
4. **验证恢复** → 健康检查
5. **事后复盘** → 记录问题

## 运维脚本

### 脚本说明

| 脚本 | 功能 |
|------|------|
| `deploy.sh` | 一键部署 |
| `health_check.sh` | 健康检查 |
| `start.sh` | 启动服务 |
| `stop.sh` | 停止服务 |
| `restart.sh` | 重启服务 |
| `logs.sh` | 查看日志 |
| `backup.sh` | 数据库备份 |
| `restore.sh` | 数据恢复 |
| `clean.sh` | 环境清理 |

### 使用示例

```bash
# 完整部署流程
./deploy/deploy.sh

# 查看所有日志
./deploy/logs.sh all 200 true

# 备份数据库
./deploy/backup.sh

# 恢复数据库
./deploy/restore.sh backups/emotion_ai_20260411.sql.gz

# 清理环境
./deploy/clean.sh
```
