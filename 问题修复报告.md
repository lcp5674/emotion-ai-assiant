## 📝 修复详情

### 问题1: 心情等级枚举 `.upper()` 错误

**位置**: `backend/app/services/diary_service.py:196`

**问题描述**:
- `_get_mood_level()` 方法已经返回正确格式的字符串（如 `"very_happy"`）
- 但代码错误地调用了 `.upper()` 方法，导致值变成 `"VERY_HAPPY"`
- 这会导致数据库验证失败，因为枚举值不匹配

**修复前**:
```python
mood_level_str = self._get_mood_level(request.mood_score)
update_data["mood_level"] = mood_level_str.upper()  # ❌ 错误
```

**修复后**:
```python
mood_level_str = self._get_mood_level(request.mood_score)
update_data["mood_level"] = mood_level_str  # ✅ 正确
```

**影响范围**:
- ✅ 日记更新功能（`PUT /api/v1/diary/{id}`）
- ✅ 心情分数自动转换为等级

---

### 问题2: WebSocket消息类型不匹配

**位置**: `backend/app/websocket/chat.py:240`

**问题描述**:
- 后端WebSocket发送的消息类型为 `"content"`
- 前端期望的消息类型为 `"chunk"`
- 这会导致流式对话无法正常显示，用户只能看到空白消息

**修复前**:
```python
await websocket.send_text(json.dumps({
    "type": "content",  # ❌ 前端不识别
    "content": chunk,
}))
```

**修复后**:
```python
await websocket.send_text(json.dumps({
    "type": "chunk",  # ✅ 前端期望的类型
    "content": chunk,
}))
```

**影响范围**:
- ✅ WebSocket流式对话功能
- ✅ 实时AI助手回复展示
