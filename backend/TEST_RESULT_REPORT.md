## 四、发现的问题

### 问题1: SQLite日期类型转换错误
**位置**: `diary_service.py` create_diary函数  
**现象**: 创建日记时，传入str类型的date，但SQLAlchemy模型中date字段是Date类型，期望date对象，导致SQLite绑定错误  
**影响**: 无法创建日记，影响日记相关所有测试  
**修复建议**: 在`diary_service.create_diary`中将date字符串转换为date对象：
```python
from datetime import datetime
diary.date = datetime.strptime(diary_create.date, "%Y-%m-%d").date()
```

### 问题2: pytest-asyncio 版本兼容性
pytest 8.0与pytest-asyncio最新版本有警告，但不影响测试运行
